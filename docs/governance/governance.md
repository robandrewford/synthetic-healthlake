# Project Governance

## 1. Purpose
This document defines how decisions are made in the FHIR-OMOP Synthetic Stack project.

## 2. Roles
### Maintainers
- Responsible for code quality, architectural decisions, and releases.
- Can approve and merge pull requests.

### Contributors
- Anyone submitting issues, documentation, or code contributions.

### Users
- Developers and analysts using the repository without contributing directly.

## 3. Decision Making
- Minor decisions: Maintainers decide and merge.
- Major decisions: Require approval from at least two maintainers.
- Breaking changes: Must be documented in DESIGN_DECISIONS.md and CHANGELOG.md.

## 4. Conflict Resolution
1. Discuss in a GitHub issue.
2. If unresolved, escalate to maintainers.
3. Final decision made by consensus of maintainers.

## 5. Release Ownership
- Only maintainers may cut releases and tag versions.
